# 辩论赛计时器 (Debate Competition Timer) - 规格文档

## 1. 项目概述

- **项目名称**: 辩论赛计时器 (DCT)
- **项目类型**: 桌面应用 (Tauri + Vue 3)
- **核心功能**: 支持自定义辩论赛模式的计时器，可存储和导出模板，配置界面与计时界面分离
- **目标用户**: 辩论赛组织者、裁判、参赛者

## 2. UI/UX 规格

## 3. 功能规格

### 3.1 核心功能

#### 模板管理
- **创建模板**: 新建空白模板
- **编辑模板**: 修改现有模板
- **删除模板**: 删除不需要的模板
- **导入模板**: 从 JSON 文件导入模板
- **导出模板**: 导出模板为 JSON 文件

#### 辩论模式配置
- **模式名称**: 自定义辩论模式名称
- **环节配置**:
  - 环节名称 (字符串)
  - 环节类型 (普通环节/自由辩论)
  - 时长 (分:秒)
  - 角色/身份 (正方一辩、反方二辩等，普通环节)
  - 正/反方标识

#### 自由辩论功能
- **双方计时**: 正方和反方各自独立计时
- **发言方切换**: 点击按钮或按 Tab 键切换发言方
- **时间同步**: 双方共用设定的总时长

#### 计时功能
- **启动计时**: 开始当前环节计时
- **暂停/继续**: 暂停或继续当前计时
- **上一环节**: 跳转到上一环节
- **下一环节**: 跳转到下一环节
- **终止计时**: 结束整个计时

#### 提醒功能
- **超时提醒**: 剩余30秒变橙色，结束时变红色闪烁
- **环节切换**: 语音/音效提示 (可选)

### 3.2 数据结构

#### 模板结构 (JSON)
```json
{
  "id": "uuid",
  "name": "标准辩论模式",
  "createdAt": "2024-01-01T00:00:00Z",
  "updatedAt": "2024-01-01T00:00:00Z",
  "segments": [
    {
      "id": "uuid",
      "name": "正方立论",
      "speaker": "正方一辩",
      "duration": 180,
      "side": "positive"
    }
  ]
}
```

### 3.3 存储方案
- 使用 localStorage 存储模板数据
- 模板数据结构见上方

### 3.4 边界情况处理
- **空模板**: 禁止开始计时，提示用户添加环节
- **时长为0**: 允许但会跳过该环节
- **导入无效文件**: 显示错误提示
- **全屏退出**: 计时继续，可通过窗口切换回配置界面
